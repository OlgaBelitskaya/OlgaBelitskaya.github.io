To encode a sequence consisting of the letters A, B, C, D, and E, an unequal binary code is used that uniquely decodes the resulting binary sequence. Here's the code:

A-110, B-10, C-01, D-00, E-111.

Is it possible to shorten the length of a codeword for one of the letters so that the code can still be decoded uniquely? Codes of other letters should not change.

Write down the answer in the form: a letter, a code word. For example, if the answer was "for the letter F the new code will be 110", the answer should be written as: F110. If it is impossible to shorten anything, write down the answer as the number 0.

.

1) 0 1 : . , 01, 10, 00 .

2) 110 111 11, 11 . 110 10, .. B.

The solution uses the Shannon-Fano Algorithm.

1) The shortest codes 0 and 1 cannot be used as code words: more than one codeword has already begun with each of these symbols. Hence, 01,10, 00 cannot be shortened.

2) 110 and 111 also cannot be cut to 11, since 11 occurs at the beginning of these two words. 110 cannot be shortened to 10 because this set of symbols has already been used to encode B.

In some information system, information is encoded in binary six-digit words. During transmitting the data, their distortions are possible, so at the end of each word the seventh (control) symbol is added so that the sum of the digits of the new word, assuming the control symbol, is even. For example, to the word 101101 on the right will be added 0, and to the word 010110 - 1.

After receiving the word, it is processed. This checks the sum of its digits, including the control ones. If it is odd, this means that when the word was transmitted, a failure occurred, and it is automatically replaced by the word 0000000. If it is even, it means there was no malfunction or there was more than one malfunction. In this case, the received word does not change.
